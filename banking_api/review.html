Banking System API — Developer Review

Internal reference for developers and QA. Covers all endpoints, request/response shapes, auth requirements, and database mappings.

Base URL: /banking/api/v1 Swagger UI: /docs Auth: JWT Bearer Admin-only endpoints marked separately

Introduction

This document describes the REST API for the banking system. The API is served by a Flask / Flask-RESTX application running behind a Tyk API Gateway.

Authentication

All endpoints require a valid JWT in the Authorization header. Obtain a token by posting your API credentials to the token endpoint.

Step 1 — Get a token

POST /banking/api/v1/auth/token
Content-Type: application/json

{
  "api_key":    "bk_live_...",
  "api_secret": "bk_secret_..."
}
{
  "access_token": "eyJ...",
  "token_type":   "Bearer",
  "expires_in":   3600,
  "role":         "admin"
}

Step 2 — Use the token

GET /banking/api/v1/clients/
Authorization: Bearer eyJ...
All cURL examples below use $TOKEN to represent the access_token value.
Bash / Git Bash: export TOKEN="eyJ..."
PowerShell: $TOKEN="eyJ..."

Roles

RoleAccessCredential set
adminFull read + write access to all endpointsAdmin key/secret
readonlyGET endpoints only — write operations return 403Analytics or Audit key/secret
Tokens expire after 1 hour. Re-authenticate when you receive a 401 with "Token has expired".

Tyk Gateway

Tyk sits in front of Flask on port 8080. Flask itself listens on port 5000 but is not directly exposed. The paths /review and /doc are whitelisted in Tyk and bypass authentication at the gateway layer.

Error Codes

CodeMeaningCommon cause

Eligibility Engine

The eligibility check at GET /loan-applications/{id}/eligibility evaluates a loan application against all active rules in LoanEligibilityRules. It requires a ClientFinancialProfiles record for the applicant.

How it works

The engine starts with a score of 100. Each rule that fails deducts its ScorePenalty from the score and sets eligible = false. The final score is clamped to a minimum of 0.

Supported rule types

RuleTypeLogicSource field
min_credit_scoreFAIL if CreditScore < ThresholdValueClientFinancialProfiles.CreditScore
max_dtiFAIL if DebtToIncomeRatio > ThresholdValueClientFinancialProfiles.DebtToIncomeRatio
min_incomeFAIL if MonthlyIncome < ThresholdValueClientFinancialProfiles.MonthlyIncome
max_loan_amountFAIL if RequestedAmount > ThresholdValueLoanApplications.RequestedAmount

Example response

{
  "eligible":       false,
  "score":          65,
  "reasons": [
    "PASS [MinCreditScore]: credit score 720 >= 650",
    "FAIL [MaxDTI]: DTI ratio 0.55 > maximum 0.45",
    "PASS [MinIncome]: monthly income 8000 >= 5000"
  ],
  "credit_score":   720.0,
  "dti_ratio":      0.55,
  "monthly_income": 8000.0
}
