Authentication Flow

All /api/* endpoints require a JWT token in the Authorization header. Tokens are issued by POST /token and expire after 1 hour. The open endpoints /token, /health, /review, and /docs require no token.

Step 1 — Request a token

POST /token
Content-Type: application/json

{
  "key":    "bk_live_xxxxxxxxxxxxxxxx",
  "secret": "bk_secret_xxxxxxxxxxxxxxxx"
}

Step 2 — Use token in every request

GET /api/clients/
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

Response: 200 OK
{...}

Token response fields

FieldTypeDescription
access_tokenstringSigned JWT — place this in the Authorization header
token_typestringAlways Bearer
expires_inintegerSeconds until expiry (3600 = 1 hour)
rolestringadmin or readonly
⚠ Secrets are hashed in the database and shown only once at generation time. Store credentials.txt securely.

Roles & Policies (enforced by Tyk Gateway)

RoleAllowed HTTP MethodsRate LimitHourly QuotaUse case
adminGET, POST, PUT, DELETE1000 requests / minUnlimitedBackend services, internal tools
readonlyGET only10 requests / min100 requests / hourDashboards, reporting, QA testing
Tyk enforces policies before the request reaches Flask. A readonly token attempting a POST will receive 403 Forbidden from the gateway — Flask never sees it.

What happens in the database

ActionTable affectedDetail
Deduct from sourceAccountsBalance -= Amount on FromAccountID
